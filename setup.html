Setup

This section provides instructions on how to set up the project environment, including installation of necessary dependencies and configuration steps.

Prerequisites

Ensure you have the following installed on your system: 1. Python 3.8 or higher
2. pip (Python package installer)
3. Access to an LLM (e.g., OpenAI’s GPT-3.5 or GPT-4, or local Ollama models)
4. Optional: A web search engine for web-based tools (default is DuckDuckGo)

LLM Configuration

nanobot is a wrapper around large language models (LLMs) that provides a simple interface for building ReAct (Reasoning + Acting) agents. To use nanobot, you need to have access to an LLM, such as OpenAI’s GPT-3.5 or GPT-4. In this setup guide, we will use local Ollama models, but you can also configure nanobot to use other LLM providers.

For configuring LLM in nanobot, you need the following information:
- Model provider: The name of the LLM provider you want to use (e.g., “ollama”, “openai”, etc.).
- Model name: The specific model you want to use from the provider (e.g., “gpt-3.5-turbo”, “gpt-4”, etc.).
- API key: If your LLM provider requires authentication, you will need an API key to access the model.
- Base URL: The base URL for the LLM provider’s API, if applicable.

The list of supported LLM providers and their configuration details can be found in the nanobot documentation. Make sure to check the documentation for any specific requirements or additional configuration options for your chosen LLM provider.

Web Search Engine Configuration

Additionally, nanobot includes web-based tools that require a web search engine. By default, it uses DuckDuckGo, but you can configure it to use other search engines if needed. This configuration also requires relatively similar information, including provider name, API key (if required), and base URL.

nanobot installation

Follow the instructions in nanobot documentation to install nanobot and its dependencies. You can choose to install it using pip or by cloning the repository and installing it locally. In this setup, we will install nanobot from source within the project environment.

  1. Create a virtual environment (optional but recommended):
python -m venv venv
source venv/bin/activate  # On Windows, use `venv\Scripts\activate`
  1. Clone the nanobot repository and install it:
git clone https://github.com/HKUDS/nanobot.git
cd nanobot
pip install -e .
  1. Verify the installation by running:

Configuration Fields

1. agents settings

There are several configuration fields in the config.json file. For our experiment which only focuses on Python and CLI commands, so sections of “channels” or “api” are not relevant. Instead, let us inspect the sections for LLM configuration, which include “agents” and “providers”. Within “agents”, there is a “defaults” section which includes the default configuration for all agents. We can set the LLM provider and model details here, as well as other relevant configuration fields such as temperature, max tokens, etc. The fields of relevant to our experiment include:

  • model: The name of the provider + model you want to use (e.g., “ollama/gpt-4o”, “openai/gpt-3.5-turbo”, etc.).
  • provider: The name of the LLM provider (e.g., “ollama”, “openai”, etc.).
  • temperature: The temperature setting for the LLM, which controls the randomness of the output (lower values make the output more deterministic).
  • maxTokens: The maximum number of tokens to generate in the response.
  • contextWindowTokens: The maximum number of tokens that can be included in the context window for the LLM.

These are the main configuration fields that influence both the performance and cost of using LLM based on baseline tasks. More extensive experiment for long-term memory / customization / multi-agent collaboration will be considered in future work.

2. providers settings

In the “providers” section, nanobot supports multiple LLM providers, each with its own subsection (e.g., “openai”, “ollama”), which typically includes:

  • apiKey: The API key for authenticating with the LLM provider, if required.
  • apiBase: The base URL for the LLM provider’s API, if applicable (e.g., for OpenAI, it would be “https://api.openai.com/v1”).
  • extraHeaders: Any additional headers that need to be included in the API requests to the LLM provider.

3. tools settings

In the “tools” section, there are configuration fields for web-based tools under the “web” -> “search” key, which include:

  • provider: The name of the web search engine provider (e.g., “duckduckgo”, “google”, etc.).
  • apiKey: The API key for authenticating with the web search engine, if required.
  • baseUrl: The base URL for the web search engine’s API, if applicable.
  • maxResults: The maximum number of search results to return for each query.
  • timeout: The timeout setting for web search queries, in seconds.

These configuration fields should be the basis for our experiment setup, which aims to evaluate the performance under cost constraints of nanobot agent. The following section declares the specific configuration we will use for our benchmarking task.

Shared Experiment Configuration

For the baseline experiment, we will use the following shared configuration for all agents:

Certain variables such as LLM_API_KEY and WEB_SEARCH_API_KEY are read from the .env file (secrets). Configure the .env file with the appropriate values for your LLM provider and web search engine. For example:
